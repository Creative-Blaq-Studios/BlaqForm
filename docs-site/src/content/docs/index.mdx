import { Card, CardGrid } from '@astrojs/starlight/components';

## Why BlaqForm?

<CardGrid stagger>
  <Card title="Zero Boilerplate" icon="rocket">
    `BfFormBuilder` eliminates manual controller creation and disposal. Declare fields as data, get a working form.
  </Card>
  <Card title="Composable Validation" icon="approve-check">
    Chain validators with `.and()`, `.or()`, `.when()`. Async validation with debounce. Cross-field matching built in.
  </Card>
  <Card title="Multi-Step Wizard" icon="list-format">
    `BfWizard` handles step navigation, per-step validation, and progress tracking out of the box.
  </Card>
  <Card title="State-Management Agnostic" icon="setting">
    Works with `setState`, Riverpod, Bloc, or vanilla `ChangeNotifier`. No lock-in.
  </Card>
</CardGrid>

## The Difference

```dart
// Before: 188 lines, StatefulWidget, manual controllers, manual dispose
class _SignupFormState extends State<SignupForm> {
  late final BfFormController _formController;
  late final BfFieldController<String> _nameController;
  late final BfFieldController<String> _emailController;
  // ... 5 more controllers, initState, dispose ...
}

// After: StatelessWidget, ~40 lines, zero ceremony
BfFormBuilder(
  fields: {
    'name':  BfFieldConfig.text(label: 'Name', validators: [Bf.required()]),
    'email': BfFieldConfig.email(label: 'Email', validators: [Bf.required(), Bf.email()]),
  },
  onSubmit: (values) async => await api.signup(values),
  builder: (context, form) => Column(children: [
    form.text('name'),
    form.email('email'),
    form.submitButton('Sign Up'),
  ]),
)
```
