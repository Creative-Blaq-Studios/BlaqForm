All validators are accessed through the `Bf` static namespace. Every validator accepts an optional `message` parameter to override the default error text.

## Sync validators

### String validators

| Name | Signature | Description | Example | Default message |
|---|---|---|---|---|
| `Bf.required()` | `BfValidator<T>` | Value is not null and not empty (trims strings). | `Bf.required()` | "This field is required" |
| `Bf.email()` | `BfValidator<String>` | Valid email address format. | `Bf.email()` | "Invalid email address" |
| `Bf.minLength(n)` | `BfValidator<String>` | String has at least `n` characters. | `Bf.minLength(8)` | "Must be at least 8 characters" |
| `Bf.maxLength(n)` | `BfValidator<String>` | String has at most `n` characters. | `Bf.maxLength(100)` | "Must be at most 100 characters" |
| `Bf.pattern(regex)` | `BfValidator<String>` | String matches the given `RegExp`. | `Bf.pattern(RegExp(r'^[A-Z]'), message: 'Must start with uppercase')` | "Invalid format" |
| `Bf.url()` | `BfValidator<String>` | Valid URL (http, https, ftp). | `Bf.url()` | "Invalid URL" |
| `Bf.phone()` | `BfValidator<String>` | Valid phone number (digits, spaces, dashes, parens, optional leading +, 7+ digits). | `Bf.phone()` | "Invalid phone number" |
| `Bf.creditCard()` | `BfValidator<String>` | Valid credit card number via Luhn algorithm. | `Bf.creditCard()` | "Invalid credit card number" |

### Numeric validators

| Name | Signature | Description | Example | Default message |
|---|---|---|---|---|
| `Bf.min(n)` | `BfValidator<num>` | Value is at least `n`. | `Bf.min(0)` | "Must be at least 0" |
| `Bf.max(n)` | `BfValidator<num>` | Value is at most `n`. | `Bf.max(100)` | "Must be at most 100" |
| `Bf.between(min, max)` | `BfValidator<num>` | Value is between `min` and `max` (inclusive). | `Bf.between(1, 10)` | "Must be between 1 and 10" |

### Date validators

| Name | Signature | Description | Example | Default message |
|---|---|---|---|---|
| `Bf.after(date)` | `BfValidator<DateTime>` | Date is after the given date. | `Bf.after(DateTime(2024, 1, 1))` | "Must be after 2024-01-01" |
| `Bf.before(date)` | `BfValidator<DateTime>` | Date is before the given date. | `Bf.before(DateTime(2030, 1, 1))` | "Must be before 2030-01-01" |
| `Bf.age(years)` | `BfValidator<DateTime>` | Date represents an age of at least `years`. | `Bf.age(18)` | "Must be at least 18 years old" |

### Generic validators

| Name | Signature | Description | Example | Default message |
|---|---|---|---|---|
| `Bf.equals(expected)` | `BfValidator<T>` | Value equals `expected`. | `Bf.equals<bool>(true, message: 'Must accept')` | "Value must equal \{expected\}" |
| `Bf.custom(fn)` | `BfValidator<T>` | Custom sync validation callback. Return `null` for valid, `BfValidationResult` for invalid. | `Bf.custom<String>((v) => v?.contains('bad') == true ? BfValidationResult('Forbidden') : null)` | (your message) |

### Cross-field validators

| Name | Signature | Description | Example | Default message |
|---|---|---|---|---|
| `Bf.matchFields(name)` | `BfValidator<T>` | Value equals the value of another field. Requires `BfValidationContext`. | `Bf.matchFields('password', message: 'Passwords must match')` | "Fields do not match" |

## Async validators

| Name | Signature | Description | Example | Default message |
|---|---|---|---|---|
| `Bf.unique(checker)` | `BfAsyncValidator<T>` | Checks uniqueness via async callback. `checker` returns `true` if unique (valid). | `Bf.unique<String>((v) => api.isAvailable(v!))` | "This value is already taken" |

## Composition operators

All `BfValidator<T>` instances support these methods:

| Method | Description | Example |
|---|---|---|
| `.and(other)` | Both must pass. Short-circuits on first failure. | `Bf.required().and(Bf.email())` |
| `.or(other)` | At least one must pass. Returns second error if both fail. | `Bf.email().or(Bf.url())` |
| `.when(predicate)` | Only runs when `predicate` returns `true`. | `Bf.required().when((ctx) => ctx?.sibling<String>('type') == 'business')` |

## Validation result params

Some validators include `params` in their `BfValidationResult` for template interpolation:

| Validator | Params |
|---|---|
| `minLength` | `{'min': n, 'actual': length}` |
| `maxLength` | `{'max': n, 'actual': length}` |
| `pattern` | `{'pattern': regexPattern}` |
| `min` | `{'min': n, 'actual': value}` |
| `max` | `{'max': n, 'actual': value}` |
| `between` | `{'min': min, 'max': max, 'actual': value}` |
| `after` | `{'date': isoString, 'actual': isoString}` |
| `before` | `{'date': isoString, 'actual': isoString}` |
| `age` | `{'min': years, 'actual': calculatedAge}` |
| `equals` | `expected`, `actual` |
| `matchFields` | `{'otherField': fieldName}` |

Use `BfMessageFormatter.format()` to interpolate these params into custom message templates.
