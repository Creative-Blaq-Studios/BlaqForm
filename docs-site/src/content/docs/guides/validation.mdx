Validators in BlaqForm are objects, not callbacks. This design enables composition, serialization, and rich error metadata. All prebuilt validators are accessed through the `Bf` namespace.

## BfValidator base class

Every synchronous validator extends `BfValidator<T>`:

```dart
abstract class BfValidator<T> {
  BfValidationResult? validate(T? value, [BfValidationContext? context]);
}
```

Return `null` for valid, or an `BfValidationResult` for invalid.

## BfValidationResult

Each validation failure carries structured data:

```dart
BfValidationResult(
  'Must be at least 8 characters',  // human-readable message
  code: 'min_length',               // machine-readable for i18n
  params: {'min': 8, 'actual': 3},  // for template interpolation
)
```

## Composition

Chain validators with `.and()`, `.or()`, and `.when()`:

### AND -- both must pass

```dart
final validator = Bf.required().and(Bf.email());
// Short-circuits: if required fails, email is not run.
```

### OR -- at least one must pass

```dart
final validator = Bf.email().or(Bf.url());
// Valid if the value is either a valid email or a valid URL.
```

### WHEN -- conditional validation

```dart
final validator = Bf.required().when((ctx) {
  return ctx?.sibling<String>('accountType') == 'business';
});
// Only validates when accountType is 'business'.
```

The `when` predicate receives an `BfValidationContext` so it can inspect sibling field values.

## Prebuilt validators

All accessed via the `Bf` static namespace:

### String validators

| Validator | Signature | Default message |
|---|---|---|
| `Bf.required()` | `BfValidator<T>` | "This field is required" |
| `Bf.email()` | `BfValidator<String>` | "Invalid email address" |
| `Bf.minLength(n)` | `BfValidator<String>` | "Must be at least n characters" |
| `Bf.maxLength(n)` | `BfValidator<String>` | "Must be at most n characters" |
| `Bf.pattern(regex)` | `BfValidator<String>` | "Invalid format" |
| `Bf.url()` | `BfValidator<String>` | "Invalid URL" |
| `Bf.phone()` | `BfValidator<String>` | "Invalid phone number" |
| `Bf.creditCard()` | `BfValidator<String>` | "Invalid credit card number" |

### Numeric validators

| Validator | Signature | Default message |
|---|---|---|
| `Bf.min(n)` | `BfValidator<num>` | "Must be at least n" |
| `Bf.max(n)` | `BfValidator<num>` | "Must be at most n" |
| `Bf.between(min, max)` | `BfValidator<num>` | "Must be between min and max" |

### Date validators

| Validator | Signature | Default message |
|---|---|---|
| `Bf.after(date)` | `BfValidator<DateTime>` | "Must be after YYYY-MM-DD" |
| `Bf.before(date)` | `BfValidator<DateTime>` | "Must be before YYYY-MM-DD" |
| `Bf.age(years)` | `BfValidator<DateTime>` | "Must be at least N years old" |

### Generic validators

| Validator | Signature | Default message |
|---|---|---|
| `Bf.equals(expected)` | `BfValidator<T>` | "Value must equal expected" |
| `Bf.custom(fn)` | `BfValidator<T>` | (your message) |

### Cross-field validators

| Validator | Signature | Default message |
|---|---|---|
| `Bf.matchFields(otherField)` | `BfValidator<T>` | "Fields do not match" |

### Async validators

| Validator | Signature | Default message |
|---|---|---|
| `Bf.unique(checker)` | `BfAsyncValidator<T>` | "This value is already taken" |

All validators accept an optional `message` parameter to override the default.

## Async validation

Async validators extend `BfAsyncValidator<T>` and return `Future<BfValidationResult?>`:

```dart
BfFieldController<String>(
  validators: [Bf.required(), Bf.email()],
  asyncValidators: [
    Bf.unique<String>((email) async {
      return await api.isEmailAvailable(email!);
    }),
  ],
  asyncDebounce: Duration(milliseconds: 500),
)
```

Async validators only run after all sync validators pass. The `asyncDebounce` parameter (default 400ms) prevents rapid-fire API calls while the user types.

During async validation, `controller.isValidating` is `true` -- use [BfFieldStatus](/guides/indicators) to show a spinner.

### Writing a custom async validator

```dart
class UsernameAvailableValidator extends BfAsyncValidator<String> {
  final ApiClient api;
  const UsernameAvailableValidator(this.api);

  @override
  Future<BfValidationResult?> validate(
    String? value, [
    BfValidationContext? context,
  ]) async {
    if (value == null || value.isEmpty) return null;
    final available = await api.checkUsername(value);
    if (available) return null;
    return BfValidationResult('Username is already taken', code: 'unique');
  }
}
```

## Cross-field validation with BfValidationContext

`BfValidationContext` provides access to sibling field values without coupling validators to the form controller:

```dart
final context = BfValidationContext(
  fieldValueGetter: <T>(name) => formController.field<T>(name).value,
);

// Inside a validator:
final password = context.sibling<String>('password');
```

The built-in `Bf.matchFields()` uses this internally:

```dart
BfFieldController<String>(
  validators: [
    Bf.required(),
    Bf.matchFields('password', message: 'Passwords must match'),
  ],
)
```

## BfMessageFormatter

Use `BfMessageFormatter` to interpolate templates with field names and params:

```dart
BfMessageFormatter.format(
  '{field} must be at least {min} characters',
  fieldName: 'Password',
  params: {'min': 8},
);
// => 'Password must be at least 8 characters'
```

You can also format directly from a validation result:

```dart
BfMessageFormatter.formatResult(
  validationResult,
  fieldName: 'Email',
);
```

Tokens use `{curly braces}`. `{field}` is replaced with the field name. Any other `{key}` is replaced with the corresponding entry in `params`. Unmatched tokens are left unchanged.

## Custom validators

### Sync

Use `Bf.custom()` for one-off validators:

```dart
Bf.custom<String>((value) {
  if (value?.contains('bad') ?? false) {
    return BfValidationResult('Contains a forbidden word');
  }
  return null;
})
```

### Extending BfValidator

For reusable validators, extend the base class:

```dart
class ContainsDigitValidator extends BfValidator<String> {
  const ContainsDigitValidator();

  @override
  BfValidationResult? validate(String? value, [BfValidationContext? context]) {
    if (value == null || value.isEmpty) return null;
    if (!value.contains(RegExp(r'\d'))) {
      return BfValidationResult(
        'Must contain at least one digit',
        code: 'contains_digit',
      );
    }
    return null;
  }
}

// Usage:
BfFieldController<String>(
  validators: [Bf.required(), const ContainsDigitValidator()],
)
```
