BlaqForm provides several widgets that enhance form UX without requiring manual state management. They all use `ListenableBuilder` internally and rebuild automatically when form or field state changes.

## BfFormProgress

Displays a `LinearProgressIndicator` reflecting how many dirty fields are currently valid:

```dart
BfFormProgress(
  controller: formController,
  showLabel: true,
  height: 4.0,
  color: Colors.green,
  backgroundColor: Colors.grey.shade200,
)
```

| Property | Type | Default | Description |
|---|---|---|---|
| `controller` | `BfFormController` | required | The form controller to track. |
| `showLabel` | `bool` | `true` | Show a "3 of 5" label below the bar. |
| `labelBuilder` | `String Function(int valid, int total)?` | `null` | Custom label builder. Receives valid and total counts. |
| `color` | `Color?` | theme primary | Filled portion color. |
| `backgroundColor` | `Color?` | `null` | Unfilled portion color. |
| `height` | `double` | `4.0` | Bar height in logical pixels. |

### Custom label

```dart
BfFormProgress(
  controller: formController,
  labelBuilder: (valid, total) => '$valid/$total fields complete',
)
```

## BfFieldStatus

Displays an icon reflecting a single field's validation state:

- **Validating** -- `CircularProgressIndicator` spinner
- **Valid + touched** -- green checkmark
- **Error + touched** -- red error icon
- **Untouched** -- empty `SizedBox`

```dart
Row(
  children: [
    Expanded(
      child: BfTextField(name: 'email', controller: emailCtrl),
    ),
    const SizedBox(width: 8),
    BfFieldStatus(controller: emailCtrl),
  ],
)
```

| Property | Type | Default | Description |
|---|---|---|---|
| `controller` | `BfFieldController` | required | The field controller to observe. |
| `size` | `double` | `20.0` | Icon/spinner size. |
| `validIcon` | `IconData?` | `Icons.check_circle` | Override valid icon. |
| `errorIcon` | `IconData?` | `Icons.error` | Override error icon. |
| `validColor` | `Color?` | `Colors.green` | Valid icon color. |
| `errorColor` | `Color?` | theme error color | Error icon color. |

## BfDirtyGuard

Wraps a widget tree in a `PopScope` that prevents navigation when the form has unsaved changes. If the user tries to leave, a confirmation dialog appears.

```dart
BfDirtyGuard(
  controller: formController,
  child: MyFormBody(),
)
```

| Property | Type | Default | Description |
|---|---|---|---|
| `controller` | `BfFormController` | required | Form controller whose `isDirty` state is monitored. |
| `child` | `Widget` | required | The widget subtree to display. |
| `title` | `String` | `'Unsaved Changes'` | Dialog title. |
| `message` | `String` | `'You have unsaved changes...'` | Dialog body. |
| `confirmText` | `String` | `'Leave'` | Confirm button label. |
| `cancelText` | `String` | `'Stay'` | Cancel button label. |

### Full page example

```dart
class EditProfilePage extends StatelessWidget {
  final BfFormController formController;

  const EditProfilePage({super.key, required this.formController});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Edit Profile')),
      body: BfDirtyGuard(
        controller: formController,
        title: 'Discard changes?',
        message: 'Your edits will be lost if you leave now.',
        confirmText: 'Discard',
        cancelText: 'Keep Editing',
        child: Padding(
          padding: const EdgeInsets.all(24),
          child: Column(
            children: [
              BfFormProgress(controller: formController),
              const SizedBox(height: 16),
              // ... form fields ...
            ],
          ),
        ),
      ),
    );
  }
}
```

## BfSubmitButton

A submit button that integrates with the nearest `BfForm` ancestor. It automatically:

- **Starts disabled** on fresh forms — fields with validators must be touched and pass before the button enables.
- Disables when any validated field has an error.
- Disables during submission to prevent double-taps.
- Shows a loading spinner while the `onSubmit` future is running.

:::note[Default behavior]
With `disableWhenInvalid: true` (the default), the button stays disabled until every field that has validators is **dirty** (user has entered a value) **and** passes validation. Fields without validators are always considered OK and don't block the button.

Set `disableWhenInvalid: false` to make the button always tappable — `submit()` will still validate before calling your callback.
:::

```dart
BfSubmitButton(
  label: 'Save Changes',
  disableWhenInvalid: true,
  onSubmit: (controller) async {
    await controller.submit((values) async {
      await api.updateProfile(values);
    });
  },
)
```

| Property | Type | Default | Description |
|---|---|---|---|
| `onSubmit` | `Future<void> Function(BfFormController)` | required | Called with the form controller on press. |
| `label` | `String?` | `'Submit'` | Button text (ignored if `child` is set). |
| `child` | `Widget?` | `null` | Custom button content. |
| `loadingWidget` | `Widget?` | small spinner | Custom widget shown during submission. |
| `disableWhenInvalid` | `bool` | `true` | Disable until all validated fields are dirty and passing. |
| `style` | `ButtonStyle?` | `null` | Passed to the underlying `ElevatedButton`. |

### Pre-filled forms (edit screens)

On edit screens where fields start with existing data, the button would stay disabled because the fields are pristine. Call `markDirty()` on pre-populated fields so they count as validated:

```dart
final email = BfFieldController<String>(
  initialValue: existingUser.email, // pre-filled from API
  validators: [Bf.required(), Bf.email()],
);
email.markDirty(); // runs validators and marks as proven

final bio = BfFieldController<String>(
  initialValue: existingUser.bio, // optional, no validators
);
// No markDirty() needed — fields without validators are always OK
```

`markDirty()` runs sync validators immediately on the current value, so the field either passes or shows an error — either way, it's no longer "unproven."

### Custom loading widget

```dart
BfSubmitButton(
  label: 'Submit',
  loadingWidget: Row(
    mainAxisSize: MainAxisSize.min,
    children: [
      SizedBox(
        width: 16, height: 16,
        child: CircularProgressIndicator(strokeWidth: 2, color: Colors.white),
      ),
      SizedBox(width: 8),
      Text('Saving...'),
    ],
  ),
  onSubmit: (controller) => controller.submit((values) async {
    await api.save(values);
  }),
)
```

## Combining indicators

A complete form with all indicator widgets working together:

```dart
BfFormBuilder(
  fields: {
    'name': BfFieldConfig<String>.text(
      label: 'Name',
      validators: [Bf.required()],
    ),
    'email': BfFieldConfig<String>.email(
      label: 'Email',
      validators: [Bf.required(), Bf.email()],
      asyncValidators: [
        Bf.unique<String>((e) => api.isEmailAvailable(e!)),
      ],
    ),
  },
  onSubmit: (values) async => await api.register(values),
  builder: (scope) {
    final emailCtrl = scope.controller<String>('email');

    return Column(
      children: [
        BfFormProgress(controller: scope.formController),
        const SizedBox(height: 24),
        scope.text('name'),
        const SizedBox(height: 16),
        Row(
          children: [
            Expanded(child: scope.email('email')),
            const SizedBox(width: 8),
            BfFieldStatus(controller: emailCtrl),
          ],
        ),
        const SizedBox(height: 24),
        scope.submitButton('Register'),
      ],
    );
  },
)
```
