BlaqForm includes a structured, colorized logger that helps you understand what is happening inside your forms at runtime. It is silent by default and configured through the `BfLogger.instance` singleton.

## Quick start

```dart
// Enable debug logging for everything
BfLogger.instance.level = BfLogLevel.debug;

// Only show validation and submission logs
BfLogger.instance.enabledCategories = {
  BfLogCategory.validation,
  BfLogCategory.submission,
};

// Silence all logging
BfLogger.instance.level = BfLogLevel.none;
```

## Configuration

All configuration is done on `BfLogger.instance`:

| Property | Type | Default | Description |
|---|---|---|---|
| `level` | `BfLogLevel` | `none` | Minimum severity to emit. |
| `enabledCategories` | `Set<BfLogCategory>?` | `null` (all) | Which subsystem categories to include. `null` means all. |
| `handler` | `BfLogHandler?` | `null` | Custom handler called before default output. Return `true` to suppress console output. |
| `useColors` | `bool` | `true` | Whether to use ANSI color codes. Set to `false` for CI or Flutter DevTools. |

## BfLogLevel

Levels are ordered from most to least severe. Setting the logger to a level enables that level and all levels above it.

| Level | Value | What gets logged |
|---|---|---|
| `none` | 0 | Nothing. |
| `error` | 1 | Errors that prevent normal operation (e.g., validation crash). |
| `warning` | 2 | Potentially harmful situations (e.g., field registered twice). |
| `info` | 3 | Key lifecycle events (e.g., form submitted, field registered). |
| `debug` | 4 | Fine-grained development messages (e.g., value changed, sync validation ran). |
| `verbose` | 5 | Very detailed output including async token tracking and debounce scheduling. |

## BfLogCategory

Filter logs to only the subsystems you care about:

| Category | What it covers |
|---|---|
| `field` | Field value changes, dirty/touched state transitions. |
| `validation` | Sync and async validator runs, results, composition. |
| `form` | Field registration, unregistration, form aggregate state. |
| `submission` | Submit lifecycle: start, validation gate, success/failure. |
| `reset` | Field reset, form reset, error clearing. |

### Example: only validation logs

```dart
BfLogger.instance
  ..level = BfLogLevel.debug
  ..enabledCategories = {BfLogCategory.validation};
```

## Console output format

Each log entry is printed as:

```
HH:MM:SS.mmm  LVL  [CATEG] fieldName -> message -- detail
```

With colors enabled, levels and categories are color-coded:

- **ERR** -- red bold
- **WRN** -- yellow bold
- **INF** -- green bold
- **DBG** -- cyan
- **VRB** -- gray

## Custom handler

Intercept log entries for analytics, crash reporting, or custom formatting:

```dart
BfLogger.instance.handler = (entry) {
  // Send to analytics
  myAnalytics.track('blaq_form_${entry.category.name}', {
    'level': entry.level.name,
    'message': entry.message,
    'field': entry.field,
    'detail': entry.detail,
  });

  // Return true to suppress default console output
  // Return false to also print to console
  return false;
};
```

### BfLogEntry

The handler receives an `BfLogEntry` with these properties:

| Property | Type | Description |
|---|---|---|
| `level` | `BfLogLevel` | Severity of this entry. |
| `category` | `BfLogCategory` | Which subsystem produced it. |
| `message` | `String` | Human-readable summary. |
| `field` | `String?` | The field name this entry relates to. |
| `detail` | `String?` | Extra detail (validation error, new value, etc.). |
| `timestamp` | `DateTime` | When the entry was created. |

## Convenience methods

`BfLogger.instance` provides shorthand methods for each level:

```dart
final logger = BfLogger.instance;

logger.error(BfLogCategory.validation, 'Validator threw', field: 'email');
logger.warning(BfLogCategory.form, 'Field registered twice', field: 'name');
logger.info(BfLogCategory.submission, 'Form submitted');
logger.debug(BfLogCategory.field, 'Value changed', field: 'email', detail: 'new@example.com');
logger.verbose(BfLogCategory.validation, 'Async token issued', field: 'username');
```

## Development setup example

```dart
void main() {
  // Enable verbose logging during development
  BfLogger.instance
    ..level = BfLogLevel.verbose
    ..useColors = true;

  runApp(const MyApp());
}
```

## Production setup example

```dart
void main() {
  // Only capture errors and warnings, send to analytics
  BfLogger.instance
    ..level = BfLogLevel.warning
    ..useColors = false
    ..handler = (entry) {
      Sentry.captureMessage(entry.toString());
      return true; // suppress console output in production
    };

  runApp(const MyApp());
}
```
